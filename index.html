🛠️ การแก้ปัญหา
❌ HTTP 404 จาก Railway

• Server ยังไม่ได้ deploy — ตรวจสอบ Railway dashboard ว่า build สำเร็จหรือยัง

• URL ผิด — Railway URL ต้องลงท้ายด้วย .up.railway.app ไม่ใช่ .railway.app

• ไม่มี railway.json ใน repo — ตรวจสอบว่าอัปโหลดไฟล์ครบ

• ทดสอบด้วย: เปิด [URL]/health ในเบราว์เซอร์ตรงๆ

❌ HTTP 401 Unauthorized

• API Key ผิด — ตรวจสอบ Railway Variables → API_KEY

• Key ที่กรอกในช่องด้านบนต้องตรงกับ Railway API_KEY ทุกตัวอักษร

• ทดสอบด้วย curl: curl -H "X-API-Key: YOUR_KEY" [URL]/api/quote?symbols=AAPL

⚠️ Claude AI "Failed to fetch"

• เป็นเรื่องปกติเมื่อเปิดใน claude.ai — browser บล็อก direct call ไป Anthropic API (CORS)

• แก้: เพิ่ม endpoint /api/ai บน Railway backend ให้ทำหน้าที่เป็น proxy แทน

• ในระหว่างนี้: AI วิเคราะห์จะทำงานได้ปกติเมื่อเปิด HTML จาก Netlify หรือ localhost

⚠️ allorigins / proxy Timeout

• allorigins.win อาจช้าหรือล่มในบางช่วง — ระบบใช้เป็น fallback เท่านั้น

• ถ้า Railway backend ทำงานปกติ จะไม่ใช้ proxy เลย

• ถ้า proxy ล่มพร้อมกับ Railway → ข้อมูลจะแสดง "—" แต่ dashboard ยังเปิดได้

📋 Railway Deploy Checklist

☐ Push ไฟล์ทั้งหมดขึ้น GitHub (server.js, package.json, railway.json)

☐ Railway → New Project → Deploy from GitHub repo

☐ ตั้ง Variables: API_KEY, CACHE_TTL_MS=60000, RATE_LIMIT=120

☐ รอ build เสร็จ (สังเกต log ว่า "TradeRadar API Server v1.0")

☐ คัดลอก Railway URL และวางในช่องด้านบน

☐ กด "ทดสอบทั้งหมด" ให้ผ่าน 6/8 ขึ้นไป

💡 ทดสอบ Railway ด้วย curl (copy แล้วรันใน terminal)

⚠️ ข้อมูลเพื่อการศึกษาเท่านั้น ไม่ใช่คำแนะนำการลงทุน
